# QFC AI Cards — AI Strategy Card Game

## Overview
A 1v1 turn-based strategy card game where cards are generated by AI inference on QFC chain. Players build decks and battle. All game state is transparent and verifiable.

## MVP Scope (v0.1)

### Card System
- **5 elements**: Fire 🔥, Water 💧, Earth 🌿, Lightning ⚡, Shadow 🌑
- **Card attributes**: name, element, attack (1-10), defense (1-10), cost (1-5), ability (text), flavorText
- **30 starter cards** (6 per element, pre-generated)
- **Card generation**: Server generates cards using weighted random with element-based rules (later: QFC AI inference)

### Battle System (1v1 Turn-Based)
- Each player starts with 20 HP and 5 mana
- Mana increases by 1 each turn (max 10)
- Draw 1 card per turn (hand limit: 7)
- Play cards by paying mana cost
- Cards attack opponent's cards or face (HP)
- **Element advantage**: Fire > Earth > Lightning > Water > Fire, Shadow neutral
- Element advantage = +2 damage
- First to reduce opponent HP to 0 wins

### Game Flow
1. Matchmaking: Enter name, wait for opponent (or play vs AI)
2. Deck selection: Pick 15 cards from your collection
3. Battle: Alternating turns until one player reaches 0 HP
4. Results: Winner gets XP, leaderboard updates

### AI Opponent
- Simple AI: plays highest attack card that it can afford each turn
- Targets weakest opposing card or face if no cards on field
- Later: AI inference via QFC chain for smarter play

### Web UI
- **Tech**: Vite + TypeScript + HTML Canvas (no Phaser, keep it light)
- **Layout**: 
  - Top: opponent's field + HP/mana
  - Middle: battlefield (cards placed here)
  - Bottom: your hand + HP/mana
  - Right sidebar: game log, turn info
- **Card design**: Programmatic (colored rectangles with element emoji, stats text)
- **Animations**: Card play (slide up), attack (flash), damage (shake), death (fade)

### Server
- Express + WebSocket on port 3220
- Room-based matchmaking
- Game state managed server-side (authoritative)
- State broadcast via WebSocket
- REST API for card collection, leaderboard

### Data Model
```typescript
interface Card {
  id: string;
  name: string;
  element: "fire" | "water" | "earth" | "lightning" | "shadow";
  attack: number;
  defense: number;
  cost: number;
  ability: string;
  flavorText: string;
}

interface Player {
  name: string;
  hp: number;
  mana: number;
  maxMana: number;
  hand: Card[];
  field: Card[];  // max 4 cards on field
  deck: Card[];
}

interface GameState {
  id: string;
  players: [Player, Player];
  turn: number;
  currentPlayer: 0 | 1;
  phase: "waiting" | "playing" | "finished";
  winner: string | null;
  log: string[];
}
```

## Constraints
- All programmatic rendering (no image assets)
- Cards are colored rectangles with text (element color-coded)
- Keep server authoritative (client sends actions, server validates)
- Support both PvP (2 browser tabs) and PvE (vs AI)
- Port 3220 to avoid conflict with qfc-office (3210)

## Card Element Colors
